# V1.4 PR Review Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a one-shot `vanguard review-pr` command that reviews an existing GitHub PR and posts a non-blocking GitHub review comment.

**Architecture:** Keep GitHub IO and model review separate. `src/runners/pr-review.ts` parses/fetches PR metadata + diff, builds the adversarial review prompt, calls an injected reviewer, strips the completion marker, and posts `gh pr review --comment`. `src/cli/review-pr.ts` adapts that pure runner to the real sandbox/agent stack using `prepareContext` + `runAgent(copyBack:false)`.

**Tech Stack:** TypeScript, NodeNext ESM, Vitest, GitHub CLI, existing Docker sandbox + AgentProvider abstractions.
