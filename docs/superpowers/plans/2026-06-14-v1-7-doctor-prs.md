# PR Watch Doctor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `vanguard doctor-prs` so operators can preflight the PR review loop before `watch-prs` claims any pull request.

**Architecture:** Reuse the existing preflight report/check formatting and add a PR-watch specific command shape. The PR doctor validates Node, LLM auth, repo remote, Docker daemon/image, GitHub auth, and the three PR review labels (`--label`, `--reviewing-label`, `--reviewed-label`).

**Tech Stack:** TypeScript, Vitest, GitHub CLI (`gh`), existing CLI parser and preflight helpers.
