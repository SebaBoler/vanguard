# Subsystem 0.5 — Sidecar Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans. Steps use `- [ ]`.

**Goal:** Harden the `vanguard __sidecar` so a desktop run UI can build on it: `repoPath` targeting, out-of-band cancel, run-id-tagged + buffered events with re-attach, non-blocking capabilities.

**Architecture:** Obey the spec's concurrency model — cancel is an OS signal (SIGUSR1), not a stdio message; runId + terminal outcome travel as events; the Rust event buffer lives behind a lock *separate* from the run-pipe mutex.

**Tech Stack:** TypeScript (strict, ESM, Node 24+), Vitest, Rust/Tauri 2, `serde_json`.

## Global Constraints

- Node `>=24`; ESM with explicit `.js` extensions on relative imports.
- **CLI additive-only.** No existing flag/behavior changed. `PROVIDER_NAMES` unchanged.
- **Never modify `.github/workflows/`.**
- `pnpm typecheck` + `pnpm test` + desktop `tsc`/tests + `cargo build`/`clippy` green before done.
- `onEvent`/`signal` undefined ⇒ byte-identical CLI.
- No `Math.random`/`Date.now` for the runId (Rust owns it; a counter).

```ts
/**
 * Current-run cancellation for the sidecar. Cancel is out-of-band: the desktop sends the sidecar
 * process a SIGUSR1 (an in-band stdio message would queue behind the run it must cancel), and the
 * handler aborts the run's controller WITHOUT exiting, so the loop stays alive for the next run.
 * Single-in-flight ⇒ one current controller at a time.
 */
let current: AbortController | undefined;

/** Mint a fresh controller for a starting run and make it current. Returns its signal for the runner. */
export function beginRun(): AbortSignal {
  current = new AbortController();
  return current.signal;
}

/** Clear the current controller once a run ends (so a stray signal can't abort the next run's setup). */
export function endRun(): void {
  current = undefined;
}

/** Abort the current run, if any. No-op when idle. Called by the SIGUSR1 handler. */
export function cancelCurrent(): void {
  current?.abort();
}

/** Install the SIGUSR1 → cancelCurrent handler once. The process does NOT exit on the signal. */
export function installCancelHandler(): void {
  process.on('SIGUSR1', cancelCurrent);
}
```

### Task 4: Rust — runId, separate buffer/active locks, envelope, cancel, backlog, repo-ok

**Files:** Modify `apps/desktop/src-tauri/src/sidecar.rs`, `apps/desktop/src-tauri/src/lib.rs`.

- [ ] **Step 1: Restructure `Sidecar` state — three separate locks**

Replace `pub struct Sidecar(pub Mutex<Option<SidecarProc>>);` with:

```rust
use std::collections::HashMap;
use std::sync::atomic::{AtomicU64, Ordering};

/// Sidecar state. `proc` (the run pipe) is held for a whole run; `buffer`/`active` use SEPARATE
/// locks so re-attach (api_run_backlog / api_active_run) reads them WHILE a run holds `proc`.
#[derive(Default)]
pub struct Sidecar {
    proc: Mutex<Option<SidecarProc>>,
    buffer: Mutex<HashMap<String, Vec<serde_json::Value>>>,
    order: Mutex<Vec<String>>, // completed-run eviction order
    active: Mutex<Option<String>>,
    counter: AtomicU64,
}
```

(Adjust `api_capabilities`/`api_create_run` to use `state.proc.lock()` where they used `state.0.lock()` — thread through a small `request(proc: &Mutex<Option<SidecarProc>>, …)` signature change.)

- [ ] **Step 2: Mint runId + emit run-accepted + `{runId,event}` envelope + buffer each event**

Rewrite `api_create_run`: mint `let run_id = format!("run-{}", state.counter.fetch_add(1, Ordering::SeqCst));`, set `*state.active.lock() = Some(run_id.clone())`, emit `run-accepted` to `api:event` as `{runId, event:{type:"run-accepted"}}` before writing the request, and in `on_event` wrap each sidecar event as `{runId, event: v["event"]}`, appending to `state.buffer` under `run_id` before `app.emit`. On terminal (result → `run-end` already in the stream; error → synthesize), append a terminal event and clear `active`.

- [ ] **Step 3: Terminal synthesis (error/cancelled)**

When `request()` returns `Err` (child EOF / error envelope), synthesize a terminal event: `{runId, event:{type:"run-error", message}}` (or `run-cancelled` if a cancel was requested for this run — track a `cancelled: bool` flag set by `api_cancel`), emit + buffer it, clear `active`. Retain last 4 completed runs in `buffer` via `order` (evict oldest).

- [ ] **Step 4: New commands**

```rust
#[tauri::command]
pub fn api_active_run(state: State<'_, Sidecar>) -> Option<String> {
    state.active.lock().ok().and_then(|g| g.clone())
}

#[tauri::command]
pub fn api_run_backlog(state: State<'_, Sidecar>, run_id: String) -> Vec<serde_json::Value> {
    state.buffer.lock().ok().and_then(|b| b.get(&run_id).cloned()).unwrap_or_default()
}

#[tauri::command]
pub fn api_cancel(state: State<'_, Sidecar>) -> Result<(), String> {
    // Out-of-band: signal the child, do not touch the proc mutex (a run holds it).
    let guard = state.proc.lock().map_err(|e| e.to_string())?;
    if let Some(proc) = guard.as_ref() {
        // SIGUSR1 to the child PID; the sidecar's handler aborts the current run.
        let pid = proc.child.id();
        // (Rust: use libc::kill(pid as i32, libc::SIGUSR1) or std nix; keep the dep minimal.)
        // ...send signal...
    }
    Ok(())
}

#[tauri::command]
pub fn api_repo_ok(repo_path: String) -> bool {
    std::process::Command::new("git").arg("-C").arg(&repo_path).arg("rev-parse")
        .arg("--is-inside-work-tree").output().map(|o| o.status.success()).unwrap_or(false)
}
```

> `api_cancel` locking `proc` to read the child PID would block behind a run. Instead store the child PID in a `Mutex<Option<u32>>` (or an `AtomicU32`) updated on spawn, so cancel reads it without the proc lock. Use that.
