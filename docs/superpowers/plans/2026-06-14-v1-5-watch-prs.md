# Vanguard v1.5: PR Review Watch Loop

## Goal

Add a focused `vanguard watch-prs` command that polls GitHub pull requests by label, claims one PR at a time with a review-in-progress label, runs the existing `review-pr` flow, and marks the PR as reviewed.

## Scope

- Keep issue/task `watch` semantics unchanged.
- Reuse the existing PR review runner instead of duplicating review prompt, diff fetching, or posting logic.
- Use labels as the deterministic routing and dedup mechanism:
  - trigger: `--label` (for example `ready for vanguard review`)
  - claimed: `--reviewing-label` (default `vanguard:reviewing`)
  - done: `--reviewed-label` (default `vanguard:reviewed`)
- Skip draft PRs and PRs authored by Vanguard/bot-style automation.
- On failure, restore the trigger label and remove the reviewing label so a later poll can retry.
- Keep logs simple and operator-readable:
  - `watch-prs: poll -> N ready`
  - `watch-prs owner/repo#12: claim -> reviewing`
  - `watch-prs owner/repo#12: reviewed -> marked`
  - `watch-prs owner/repo#12: failed -> retry later`

## Implementation Steps

1. Add a pure runner in `src/runners/pr-watch.ts` with injected GitHub and review primitives.
2. Add `watch-prs` parsing in `src/cli/args.ts`.
3. Add a CLI command in `src/cli/watch-prs.ts` that wires polling to `reviewPrCommand`.
4. Register the command in `src/cli/index.ts`.
5. Export the new runner APIs from `src/index.ts`.
6. Document the operator command in `README.md`.

## Tests

- Runner unit tests for filtering, label swaps, retry behavior, and logging.
- CLI parser tests for defaults and required label/repo.
- CLI command test with injected runner, no Docker or real GitHub calls.
- Focused Vitest run, then typecheck and full test suite.
